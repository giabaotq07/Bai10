1. Tại sao khi chạy GitHub Actions lại báo lỗi "Tôi không thấy file POM nào ở đây cả!"?
Error: The goal you specified requires a project to execute but there is no POM in this directory (/home/runner/work/Bai10/Bai10). Please verify you invoked Maven from the correct directory. -> [Help 1]

Error:

Error: To see the full stack trace of the errors, re-run Maven with the -e switch.

Error: Re-run Maven using the -X switch to enable full debug logging.

Error:

Error: For more information about the errors and possible solutions, please read the following articles:

Error: [Help 1] http://cwiki.apache.org/confluence/display/MAVEN/MissingProjectException

Error: Process completed with exit code 1.
Khi bạn nhấn "Push" code lên GitHub, một máy ảo (Ubuntu) sẽ được khởi tạo để chạy các lệnh trong file ci.yml.
Tuy nhiên, máy ảo này ban đầu là trống rỗng.
Mặc dù code của bạn đã nằm trên Server của GitHub, nhưng bạn chưa ra lệnh cho máy ảo tải
(clone) code đó về thư mục làm việc của nó. Khi lệnh mvn package chạy, nó tìm file pom.xml
để biết cách build, nhưng vì thư mục đang trống nên nó báo lỗi: "Tôi không thấy file POM nào ở đây cả!".
Bạn cần thêm bước Checkout vào trước bước cài đặt Java. Đây là bước "mở cửa" để máy ảo kéo code từ repository về.

Maven cố gắng tải thư viện Logback phiên bản 9.9.9 từ kho lưu trữ trực tuyến (Maven Central).
Tuy nhiên, trên thực tế, phiên bản cao nhất hiện tại của Logback chỉ mới loanh quanh đầu 1.x hoặc 1.4.x.
Phiên bản 9.9.9 là một con số "ảo" không tồn tại, khiến Maven không thể tải về để build dự án.
Bạn cần mở file pom.xml và sửa lại số phiên bản của logback-classic về một phiên bản thực tế.

3 Tại sao khi chạy GitHub Actions lại không chạy qua 1 test nào cả?
Dấu hiệu: Pipeline báo xanh nhưng thực tế 0 test nào được thực thi.
Nguyên nhân kỹ thuật: * Trong pom.xml, bạn đang dùng thư viện Test là JUnit 5 (junit-jupiter).
Tuy nhiên, plugin dùng để chạy test là maven-surefire-plugin lại đang ở phiên bản 2.12.4.Sự thật là
Các phiên bản Surefire cũ (dưới 2.22.0) không hề biết JUnit 5 là gì,
nên nó đi ngang qua các file Test của bạn mà... không làm gì cả. Nó tưởng dự án không có test nên báo "Xong!"
một cách vui vẻ.Bạn cần cập nhật maven-surefire-plugin lên phiên bản mới hơn trong file pom.xml.
